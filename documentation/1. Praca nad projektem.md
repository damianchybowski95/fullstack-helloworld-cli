## Komendy

1. `npm run start` - Buduje projekt, instaluje go globalnie pod nazwą fullstack-cli, oraz uruchamia cli globalnie.
2. `npx fullstack-cli` - Uruchamia cli.
3. `npm run build` - Buduje projekt w folderze ./dist
4. `npm run run_` - uruchamia lokalną wersje projektu.
5. `npm run install_` - instaluje projekt globalnie pod nazwą fullstack-cli, tak że można go uruchamiać globalną komendą npx fullstack-cli.
6. `npm run test` - Wykonuje testy

## Struktura cli

Cli składa się z:

1. Status projektu - Uruchamia się regularnie by sprawdzić stan projektu. To ekran startowy cli.
   1.1. Wykrywa system operacyjny ( linux || windows ).
   1.2. Wykrywa wersje zainstalowanych i wymaganych sdk, czyli: node, npm, java.
   1.3. Wykrywa jakie projekty są już zainicjowane w folderze.
2. Sdk. - Instaluje, lub pokazuje komendy potrzebne do instalacji, lub zmiany sdk.
   2.1. Instaluje node i npm. ( używając chocolatey )
3. Init project - Inicjuje jeden z projektów w folderze. Może zainicjować:
   3.1. Dokumentacje napisaną w docosaur.
   3.2. Stronę internetową napisaną w nextjs ( statyczną )
   3.3. Stronę internetorwą napisaną w nextjs ( dynamiczną, z SSR )
   3.4. CMS, dla każego z projektów.
   3.5. Panel administracyjny.
   3.6. Backend napisany w Spring
   3.7. Backend napisany w js
   3.7. Aplikacja mobilna napisana w Expo ( react native ).
4. Manage projects - zarządza projektami, czyli wykonuje na nich powszechne operacje ( dev, start, test, build, install, deploy )
5. Common tests - Uruchamia testy zgrupowane pomiędzy projektami.
